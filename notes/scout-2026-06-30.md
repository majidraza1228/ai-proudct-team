# Scout Report — June 30, 2026

## What's New (filtered for AI Infrastructure Engineer role)

**kubernetes-sigs/agent-sandbox CRD** (March 2026)
- Official Kubernetes CRD for running AI agents in isolated sandboxes on-cluster
- Matters for: tool sandboxing + least-privilege requirement in the job
- Read: https://kubernetes.io/blog/2026/03/20/running-agents-on-kubernetes-with-agent-sandbox/

**KEDA GPU autoscaling external scaler** (CNCF, May 2026)
- KEDA now has a pattern for GPU-backed agent workload autoscaling
- Matters for: "autoscaling for bursty agent traffic" — direct job requirement
- Read: https://www.cncf.io/blog/2026/05/27/gpu-autoscaling-on-kubernetes-with-keda-building-an-external-scaler/

**LiteLLM Agent Platform** (May 2026)
- Kubernetes-native self-hosted layer for isolated agent sandboxes + persistent session management
- Shows the production pattern: agents as sandboxed, persistent, stateful K8s workloads
- Read: https://www.marktechpost.com/2026/05/16/meet-litellm-agent-platform-a-kubernetes-based-self-hosted-infrastructure-layer-for-isolated-agent-sandboxes-and-persistent-session-management-in-production/

**Amazon Bedrock AgentCore Identity**
- AWS standalone identity service scoping what each agent can access (works on EKS, ECS, Lambda)
- Matters for: "secrets and permissioning model that scopes what agents can do"
- Read: https://aws.amazon.com/blogs/machine-learning/secure-ai-agents-with-amazon-bedrock-agentcore-identity-on-amazon-ecs/

**Claude Sonnet 4 + Opus 4 retired** (June 15, 2026)
- These model IDs now return errors. Use claude-sonnet-4-6 or claude-opus-4-7
- Action: Update any local projects using old model IDs now

**MCP tunnels** (Anthropic, research preview)
- Private-network MCP server access without exposing endpoints publicly
- Relevant to Month 2 security work — bookmark now

## Direction for Month 1 (July)

**This week:** Read the agent-sandbox CRD post, set up a local kind cluster, deploy one agent using the CRD.

**Build this month:** A simple agent on Kubernetes with KEDA scaling triggered by a queue (SQS or Redis). Interview-ready artifact.

**Skip:** vLLM, KServe, Ray — those are inference serving. The job is about running agents, not serving open-weight models.

## Flags

- agent-sandbox CRD is not widely known yet — learning it now is a differentiator
- AgentCore Identity is the clearest AWS-native example of agent permissioning — study the architecture
- Most "AI on Kubernetes 2026" articles cover inference stacks, not agent ops — don't let them define your mental model
