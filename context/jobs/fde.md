# Forward Deployed Engineer (FDE)

**Added:** 2026-06-30
**Status:** Active target

## Role Overview

Forward Deployed Engineers embed directly with enterprise customers to deploy, customize, and extend AI products in the customer's environment. The role is part engineer, part consultant, part solutions architect.

Companies hiring FDE: Palantir, Scale AI, OpenAI, Anthropic, Sierra, Harvey, Glean, Cohere, Inflection, and most Series B+ AI-native companies with enterprise sales.

## What Companies Want

- Deploy AI products directly into enterprise customer environments (VPCs, on-prem, hybrid)
- Translate customer workflows into agent/LLM solutions — fast (days not weeks)
- Own the technical relationship: integrate, debug, extend
- Ship working prototypes under real constraints (enterprise IT, compliance, data residency)
- Explain AI tradeoffs to non-technical stakeholders (CISOs, ops, executives)
- Strong breadth across Claude, OpenAI, Azure, and some infra

## Must-Have Skills

- Claude + OpenAI API — build agents fast from scratch without scaffolding
- Microsoft/Azure stack — enterprise customers run on Azure; know Entra, private endpoints, managed identity
- Fast prototyping: RAG pipeline, document Q&A, workflow automation in 1–3 days
- Security: VPC, data residency, no-data-egress constraints, prompt injection awareness
- Communication: explain model tradeoffs, token costs, and prompt injection risk to non-engineers
- Customer delivery: scope, build, demo, train, hand off

## Sample Interview Questions

- "A Fortune 500 customer says our AI product doesn't fit their workflow. What do you do?"
- "You're on-site. Their IT team won't give you API access. How do you move?"
- "Build a document Q&A agent for a law firm using Claude in 3 days."
- "The customer's data can't leave their VPC. How do you run the product?"
- "Explain prompt injection risk to a CISO who's never heard of it."

## How It Overlaps with AI Infra

Skills that serve both targets:
- Claude/OpenAI/Azure API depth
- Agent architecture (tool use, multi-step, state management)
- Security (prompt injection, secrets, least privilege)
- Token optimization and cost engineering
- Kubernetes + AWS (strong FDE differentiator — enterprise customers have complex infra)
- Observability (debugging agents in customer environments without full access)
